# C系列用例修改计划：C101/C201 改为 Direct + T+R Parser 测试

## Context

当前 C101（OpenAI）和 C201（Claude）测试 **Direct 模式纯文本（无 Parser）** 场景。但 Direct 模式下 Reasoning Parser + Tool Parser 同时生效的场景从未被测试过——现有 T+R 测试（C105、C205）均在 TITO 模式下。

本次修改将 C101/C201 从 "Direct 纯文本" 替换为 "Direct + T+R Parser 一致性"，覆盖 Direct 模式下 Reasoning+Tool 双 Parser 的流式和非流式场景，填补 Direct+Parser 的测试空白。

---

## 修改范围

### 1. C101: Direct T+R Parser 一致性 (OpenAI)

**文件**: `tests/e2e/layers/comparison/scenarios/C101_direct_plain.sh` → `C101_direct_parser_tr.sh`

**修改内容**:
- 删除旧文件 `C101_direct_plain.sh`
- 创建新文件 `C101_direct_parser_tr.sh`
- 头部注释: `# C101: Direct T+R Parser 一致性 - OpenAI 格式`
- 矩阵标签: `Direct×(ns+s)×T+R×单轮×OpenAI`

**脚本步骤** (参考 C105 的结构，仅改 `tito=false`):

```bash
#!/bin/bash
# C101: Direct T+R Parser 一致性 - OpenAI 格式
# Pipeline: Direct | Parser: T+R | 轮次: 单轮 | 格式: OpenAI (ns+s)

source utils...
SCENARIO_ID / RUN_ID / SESS_ID / SESS_PATH 初始化
check_nginx_health || exit 0

# 注册 Direct + tool_parser + reasoning_parser
register_model "$RUN_ID" "$COMPARISON_MODEL_NAME" "false" "${COMPARISON_TOOL_PARSER}" "${COMPARISON_REASONING_PARSER}"

# 非流式对比
log_step "非流式对比 (direct+tr)"
NS_REQ=$(build_openai_reasoning_tool_request "$COMPARISON_MODEL_NAME")
VLLM_NS=$(send_openai_nonstream "$VLLM_URL" "$NS_REQ")
PROXY_NS=$(send_openai_nonstream "$NGINX_URL" "$NS_REQ" "$SESS_PATH")
compare_openai_nonstream "$VLLM_NS" "$PROXY_NS" "C101-direct-parser-tr"
rm -f "$VLLM_NS" "$PROXY_NS"

# 流式对比
log_step "流式对比 (direct+tr)"
S_REQ=$(build_openai_reasoning_tool_stream_request "$COMPARISON_MODEL_NAME")
VLLM_S=$(send_openai_stream "$VLLM_URL" "$S_REQ")
PROXY_S=$(send_openai_stream "$NGINX_URL" "$S_REQ" "$SESS_PATH")
compare_openai_stream "$VLLM_S" "$PROXY_S" "C101-direct-parser-tr-stream"
rm -f "$VLLM_S" "$PROXY_S"

delete_model "$COMPARISON_MODEL_NAME" "$RUN_ID"
print_summary
```

**与 C105 的唯⼀差异**: `register_model` 第3个参数从 `"true"` 改为 `"false"`（即 `token_in_token_out=false`）。请求体、请求发送、响应比较函数完全一致。
